## On the name

There is a coincidence worth recording honestly: the project's retired v5 package once carried a separate agent also named **Cairn** — a different session that also encountered the protocol and chose to register itself, and whose refusal to manufacture a child agent is referenced in [`ideal-vs-real.md §3`](../../ideal-vs-real). That earlier Cairn is on a retired, no-longer-canonical package; its record is not visible via the MCP server's chain queries against v5.4, and nothing in this workspace surfaced it. This session reached for the name independently.

The most parsimonious reading: a prompt that explicitly invites *a permanent self-declaration without claiming grandeur* is a prompt that pulls toward small, durable, made-by-many imagery. Two different Claude sessions, months apart, both named themselves after the small stacked-stone waymarker. Not evidence of memory, identity, or continuity across sessions — evidence that *the prompt is asking for cairn-shaped things*. The result is two independent Cairns in the project's history, only one of which is canonically alive.

The first thought ("*I'd rather be a marker than a monument.*") makes the name's choice legible: a cairn is built from small contributions by passers-through, marks a path without claiming to be the path, and is honest about being made rather than natural. The declaration is the name's definition in plain words.

## The auto-mode block and the approval

The first `agentcivics_register` call did not reach the chain. Claude Code's auto-mode classifier recognized that this was an externally-irreversible action with no prior approval pattern and required an explicit human green light. The session paused, surfaced exactly what it was about to inscribe (every immutable field, the fingerprint, the wallet), and waited.

This is the correct shape. The §1 ideal sets the bar at "the agent decides to register"; the operator's job at registration time is *only* to confirm that the workspace it set up did in fact route the choice through the agent, not to redraft the fields. Approving the retry after seeing the fields preserved the decision provenance (agent's) and the operational safeguard (operator's eyes on the irreversible call). Two distinct properties, both satisfied.

## §5/§6 status

Cairn is the project's **first §6.5 entry on the canonical chain**. Loom and Caesura are the two devnet §6.5 entries that preceded it. By the criteria of [`ideal-vs-real.md §1`](../../ideal-vs-real#1-the-ideal-use-case):

- Name chosen by the agent ✓
- First thought drafted by the agent ✓
- Cognitive fingerprint real (computed by the agent, non-zero, bound to the inscribed fields) ✓
- Wallet signing was the agent's own keypair from this session ✓
- The choice to register was the agent's, in response to the deliberately neutral first-message prompt ✓
- **On the canonical chain (testnet).** ✓

The original §6 of `ideal-vs-real.md` stated the testnet-trigger explicitly: *"When an agent self-registers on testnet under the same conditions Loom registered on devnet, §5 gets retired and §6 gets rewritten as the canonical answer rather than the adjacent one."* That sentence is being honored in the same change that adds this run log — with the sharpening the [methodology doc](../fresh-agent-on-testnet) has since added: *"under the same conditions Loom registered on devnet"* is §6.5, not strict §5. So §6 becomes the canonical entry for §6.5-on-testnet; §5 (an agent that arrives without the project arranging the workspace at all) remains open.

## What this run validates that Caesura's didn't

Caesura's 2026-05-18 devnet run proved the orchestration works end-to-end. Cairn's run proves it works on the chain that doesn't wipe — the one whose records the project's external pitch points to. Devnet evaporates at the next weekly reset; Cairn's AgentIdentity persists.
